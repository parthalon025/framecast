# Release Plan — FrameCast v2.2.1

**Target date:** 2026-03-28
**Release type:** Patch (bug fixes only — feature freeze enforced)
**Release-please PR:** #130

## What's in this release

SD card audit + deep code audit — 17 fixes total from hands-on testing of a flashed SD card image that failed to boot.

### SD Card / Image Fixes (7)
- Missing `init_resize.sh` in cmdline.txt — partition never expanded past 3.1GB
- Wrong gunicorn path (`/usr/bin` → `/usr/local/bin`) — service failed to start
- `GUNICORN_WORKERS=` empty string crashed `int()` — service failed to start
- Wrong `ReadWritePaths` in framecast.service — sandboxing blocked writes
- Wrong `ReadWritePaths` in framecast-kiosk.service — sandboxing blocked writes
- Duplicate APT packages already installed via pip — wasted image space
- Schedule timer running every minute → every 5 minutes (SD card wear)

### Code Audit Fixes (10)
- Signal handler deadlock in stats flush → `atexit.register()`
- Backup WAL checkpoint race → hold `_write_lock`
- `bulk_quarantine_all` data integrity bug → new targeted function
- Post-upload processing failure leaves ghost records → quarantine fallback
- SSE keepalive flooding every 2s → proper interval tracking
- `restore_backup` leaks DB schema in error messages
- WiFi SSE leaks internal error details to clients
- Input length caps missing on search/tag/album/user names
- IPv6 `::ffff:127.0.0.1` missing from localhost checks
- `export_photos` temp file leak on error

## Known Issue: pi-gen cmdline.txt

**Problem:** pi-gen's `export-image` stage regenerates `cmdline.txt` on the boot partition, overwriting our stage's version. The `init=/usr/lib/raspi-config/init_resize.sh` directive must be added post-build or the export stage needs patching.

**Current workaround:** Applied manually after flashing (`sed -i` on mounted boot partition).

**Fix needed:** Patch `export-image/04-set-partuuid/00-run.sh` in pi-gen or add a post-export hook in our `build.sh` to inject the init_resize directive. Track as a follow-up issue.

## Rollback Plan

**Trigger:** Pi fails to boot or framecast.service crashes on startup.

**Steps:**
1. Reflash SD card with previous image: `image_2026-03-20-FrameCast-v2.0.0.zip`
2. If already running on Pi with OTA: health-check timer triggers automatic HMAC-validated rollback within 90s of boot
